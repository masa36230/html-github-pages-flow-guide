Cursor スキル図解

HTML を作って、Git に載せて、
世界中に公開するまで

「html-to-github-pages」スキルの流れを、AI 初心者の方でも頭に入るようにビジュアル化しました。Cursor に依頼するイメージと、自分で触るポイントが分かります。

1 全体は3つの山

① 下ごしらえ

HTML を用意し、画像など同梱ファイルをまとめてフォルダに置く

② GitHub へ送る

Git で記録し、GitHub にリポジトリとして載せる

③ 公開 URL

GitHub Pages が Web サーバの役割になり、誰でも見られる URL が付く

2 誰が何をする？

Cursor（AI）が得意なこと

  • HTML の作成・修正、デザインの調整
  • フォルダ作成、index.html の保存、アセットの同梱
  • Git の初期化、コミット、公開用スクリプトの実行案内
  • 公開後の URL とローカルパスの整理・記載

あなたが一度だけ／ときどきやること

  • 初回だけ gh auth login（GitHub CLI のログイン）
  • リポジトリ名の確認（英小文字・数字・ハイフン）
  • 更新時は HTML を直したあと、push を依頼または実行

3 初回公開の流れ（ステップ）

上から順に進むイメージです。番号はスキル内の工程に対応しています。

STEP 01

HTML と「置き場所」を決める

ファイルパスで渡すか、チャットに HTML を貼る。LP ならルートに index.html。リポジトリ名は英数字の名前に。

あなた：素材・希望 AI：保存・命名の整理
STEP 02

サイト用フォルダにまとめる

マイドライブの github_pages_sites/<リポジトリ名>/ に配置。画像や CSS など相対パス参照があるものは同じフォルダへ。

AI：フォルダ作成・コピー
STEP 03

Git で「スナップショット」を1枚目撮る

git init → git add → git commit。これで「この時点のファイル一式」が記録されます。

AI：コマンド実行または案内
STEP 04

GitHub にログインできるか確認

未ログインなら gh auth login を1行だけターミナルで実行（他のコマンドと混ぜない）。

あなた：ブラウザ認証 AI：要否の確認
STEP 05

公開スクリプトでリポジトリ作成＋Pages 有効化

publish_github_pages.sh を環境変数付きで実行。リモート作成、push、Pages 設定まで一気に進みます。

AI：実行・結果の共有
完了：納品される情報の例
https://（ユーザー名）.github.io/（リポジトリ名）/

初回は数分 404 が続くことがあります。スーパーリロードも有効です。

つまずきやすいポイント
gh auth login の途中で cd や別コマンドを同じ貼り付けに混ぜると失敗しやすいです。必ず「ログイン用のコマンドだけ」を1つずつ実行してください。

4 2回目以降の更新（もっと短い）

Pages の設定が済んでいれば、公開スクリプトは不要です。push だけで更新できます。

HTML を編集 → git add / commit → git push → サイトに反映

5 Cursor にこう頼むとスムーズ

初回

「この HTML を GitHub Pages で公開して」「index.html をこの内容で作って Pages に載せて」

更新

「同じサイトの HTML を直したので push して」「公開済みリポジトリの index.html を更新して反映して」
